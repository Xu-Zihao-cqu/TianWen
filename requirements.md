# 个人作品集网站需求文档

**项目名称：天问**
**文档版本：** v0.1  
**创建日期：** 2026-06-10  
**状态：** 待确认

---

## 1. 项目概述

开发一个个人作品集展示网站，用于展示硬件项目、软件项目、资源分享及在校作业四大类内容。网站分阶段迭代开发，v1 为纯静态展示版本，后续版本逐步引入动态功能。

---

## 2. 版本规划

| 版本 | 目标 | 状态 |
|------|------|------|
| v1 | 静态作品集展示，手动更新内容 | 当前计划 |
| v2+ | 后台管理、用户评论、文件上传等动态功能 | 未来规划 |

---

## 3. 功能需求

### 3.1 页面结构

#### 首页 / 个人介绍页（Hero + About）
- 个人头像展示
- 详细个人简介
- 技能栈展示（可视化呈现，如进度条、标签云等）
- 联系方式（邮箱、GitHub、社交账号等）

#### 作品集主页
- 四大板块入口：硬件项目、软件项目、资源分享、在校作业
- 板块设计需预留扩展空间，方便未来新增板块

#### 各板块作品列表页
每个作品卡片需包含：
- 作品名称
- 封面图 / 预览图
- 简短描述
- 标签系统（支持多标签，如技术栈、类别、年份等）
- 标签筛选功能（按标签过滤作品）
- 文件类型标识（源代码、PDF、链接等）

#### 作品详情页
- 完整描述
- 标签列表
- 文件列表（支持在线预览 PDF、在线查看源代码）
- 文件下载按钮
- 外部链接（GitHub 仓库、演示地址等，按需）

### 3.2 文件托管方案（推荐）

推荐使用 **GitHub 仓库 + Git LFS** 托管方案：

- 代码文件、小型资源直接存入同一 GitHub 仓库
- PDF、较大文件使用 Git LFS 管理
- 优点：免费、版本可控、与部署流程无缝衔接
- 备选方案：若单仓库文件量过大，可将文件单独存放于另一 GitHub 仓库，通过 Raw URL 引用

### 3.3 中英双语支持
- 全站支持中文 / 英文切换
- 切换按钮置于导航栏显眼位置
- 语言偏好通过 `localStorage` 持久化保存

### 3.4 导航与通用功能
- 顶部导航栏：含各板块快速跳转、语言切换、（未来预留）登录入口
- 响应式设计：兼容桌面端、平板、手机
- 平滑滚动与页面过渡动效

## 4. 视觉设计要求

- **风格：** 活泼有创意，富有现代感
- **色彩：** 渐变色主题（如渐变 Hero 区域、渐变按钮、渐变标签）
- **动效：** 丰富但不影响性能，包括：
  - 页面滚动入场动画（Intersection Observer）
  - 卡片悬停效果（scale、shadow、glow）
  - 页面/路由切换过渡动画
  - 打字机效果（用于个人介绍标题）
- **图片资源：** 可使用 Unsplash、unDraw、Storyset 等开源图库
- **字体：** 中文推荐思源黑体（Noto Sans SC）；英文推荐 Inter 或 Space Grotesk

---

## 5. 技术栈推荐

### 5.1 前端框架
推荐 **React + Vite**：
- 组件化开发，便于后续添加动态功能
- 生态丰富（动效库、组件库选择多）
- 与 Vercel 部署无缝集成

### 5.2 推荐配套库

| 用途 | 推荐库 |
|------|--------|
| 动效 | Framer Motion |
| 样式 | Tailwind CSS |
| 国际化（i18n） | react-i18next |
| PDF 预览 | react-pdf |
| 代码高亮 | Prism.js / Shiki |
| 图标 | Lucide React |

### 5.3 部署平台
推荐 **Vercel**：
- 免费额度充足
- 自动检测 React/Vite 项目，零配置部署
- 支持 GitHub 仓库自动触发 CI/CD（推送即部署）
- 未来购买域名后，支持一键绑定自定义域名

### 5.4 域名
- v1 使用 Vercel 默认域名（如 `yourname.vercel.app`）
- 后续版本可购买自定义域名并在 Vercel 中绑定

## 6. 未来版本（v2+）功能预留

以下功能 v1 **不实现**，但技术选型和代码结构需预留扩展空间：

- **作者后台管理**：登录后可在各板块直接上传、编辑、删除作品
- **用户评论系统**：已登录用户可在作品详情页发表评论
- **用户文件上传**：已登录用户可上传附件至评论区
- **访客无需登录**：仅浏览和下载功能对所有人开放，无需注册

---

## 7. 非功能性需求

- **性能：** 首屏加载时间 < 3s（图片懒加载、代码分割）
- **稳定性：** 依托 Vercel CDN，保证高可用
- **SEO：** 静态页面需支持基础 SEO（meta 标签、OG 标签）
- **可维护性：** 作品数据以结构化 JSON/JS 文件存储，新增作品只需修改数据文件，无需改动组件代码
- **扩展性：** 板块数量可配置，方便未来新增类别
